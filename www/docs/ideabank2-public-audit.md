# Аудит публичной части `uds.ideabank2` по прототипу `/local/_/tmp/ideas`

## Цель

Публичная часть банка идей должна соответствовать прототипу не как одна SPA-страница с параметром `ib-page`, а как набор нормальных Bitrix-страниц и специализированных компонентов. Компоненты отвечают за UI и подготовку данных, бизнес-логика остаётся в модуле `uds.ideabank2`.

|---|---|---|---|

## Основные блоки прототипа

### Главная

- переключение сценария сотрудник / рабочий контур;
- hero-блок;
- «Как это работает в 3 шага»;
- метрики доверия;
- мои коины и история начислений;
- быстрые ссылки;
- рабочий фокус роли;
- очереди по роли;
- новости;
- тематические челленджи;
- CTA для руководителей;
- культура банка идей;
- голоса авторов;
- лучшие реализованные идеи;
- рейтинг лидеров.

### Реестр идей

- список идей;
- фильтр по поиску, статусу, категории, вкладке;
- режимы `all`, `mine`, `drafts`, `best`, `moderation`;
- пагинация;
- переходы к карточке и форме.

### Форма идеи

- 4 шага;
- тип, категория, бизнес-направление;
- базовая идея;
- инициатор, соавторы, доли участия;
- анонимность;
- ключевые слова;
- участок / текущее состояние;
- проблема, потери, предложение, ожидаемый результат;
- экономический эффект, дата внедрения;
- доп. проработка, нематериальные эффекты;
- план, ресурсы, риски;
- вложения;
- похожие идеи;
- сохранение черновика и отправка на модерацию.

### Карточка идеи

- основные поля идеи;
- авторы и связанные данные;
- блок эффекта и внедрения;
- обсуждение, реакции, комментарии;
- панель модератора;
- экспертные оценки;
- решение комитета;
- workflow/SLA;
- обратная связь;
- история начислений.

## Выявленные расхождения текущей реализации

1. Был один компонент `uds:ideabank`, который пытался рендерить все экраны через `?ib-page=`.
2. URL-структура не соответствовала прототипу.
3. Нет отдельных компонентных классов для страниц.
4. Данные прототипа были частично не перенесены в БД: не хватало идей, новостей, конкурсов, челленджей, комментариев, реакций, workflow, экспертных оценок, решений комитета и истории коинов.
5. Часть методов и ORM-полей не соответствовала фактическим Table-классам.

## Целевая реализация

- `/ideabank/*.php` — публичные страницы.
- `/local/components/uds/ideabank.*` — отдельные компоненты.
- `/local/js/uds/ideabank2/public.css` и `public.js` — общие assets.
- `Uds\Ideabank2\Seed\DemoDataSeeder` — идемпотентное наполнение демо-данными.
